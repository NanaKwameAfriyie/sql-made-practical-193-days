# SQL Made Practical: 193 Days of Real Business Problems

This repository documents a daily public SQL teaching journey from 22 June to 31 December 2026.

## Teaching method

Every lesson follows:

**Question → Grain → Query → Validation → Business Decision**

## Repository structure

- `day-001/` — Launch lesson: defining “best customer” precisely
- `datasets/` — Reusable datasets and data dictionaries
- `assets/` — Images, diagrams and screenshots used in lessons
- `notes/` — Planning notes and retrospective observations

## Working principles

1. Never publish a query that has not been executed and validated.
2. State what one row represents before and after every transformation.
3. Include assumptions, exclusions and at least one quality check.
4. Connect every query to a business decision.
5. Keep examples reproducible with fixed dates and deterministic sample data.
